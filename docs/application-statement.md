# Agent/AI 项目成果说明

我构建了一个代码库健康巡检工具：`repo-health-auditor`。它解决的核心痛点是：个人开发者或小团队在接手、维护、重构一个代码库时，技术债、测试缺口、README 缺失、疑似密钥、超大文件、TODO/FIXME 等问题往往分散在不同目录中，人工排查效率低，也很难快速形成可执行的治理路线图。

项目采用多 Agent 协作方式完成长链分析流程。`FileScoutAgent` 首先扫描仓库结构、语言分布、文件规模和关键配置文件；`RiskAnalystAgent` 基于扫描结果识别维护风险和安全风险，例如疑似密钥、遗留 TODO、空文件、超大文件等；`TestStrategistAgent` 进一步分析测试目录、测试文件和可推断的测试命令，判断项目是否存在测试覆盖缺口；最后 `RoadmapAgent` 汇总前序 Agent 的结果，按照高、中、低优先级生成工程改进路线图。

项目支持对任意本地代码库生成 Markdown 和 JSON 两种报告，适合用于新项目接手、重构前评估、PR 前自检和团队技术债治理。它也预留了兼容 OpenAI 接口的模型总结能力，可以在基础规则巡检结果之上接入大模型，生成更适合管理者阅读的摘要。当前项目已上传到 GitHub，可直接安装并通过 CLI 运行。
